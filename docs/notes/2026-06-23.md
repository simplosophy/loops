# 2026-06-23

## HLP 架构评审后的协议硬化

- 根据协议分层、SDK/API、runtime adapter、生产可靠性四个视角的评审，启动 HLP reference implementation hardening。
- 本轮优先修复会破坏协议可信度的问题：adapter 失败后状态不得提前推进、SDK/read API 不返回可变活对象、checkpoint resolution 做最小授权校验、事件总线遵循公开协议、handoff/cancel/delegate run binding 更一致。
- durable store v2、audit append-only hash chain、事务 outbox/recovery 属于生产化关键后续，先在计划和架构文档中明确边界，避免把 snapshot store 误描述为生产级持久层。
- 已实现 fail-before-commit：`task.assign`、`checkpoint.raise`、`checkpoint.resolve`、`ownership.transfer(handoff)`、`ownership.delegate`、`task.cancel` 在 adapter 失败时不推进 HLP 状态/audit/run binding。
- 已实现 SDK/read snapshot：`HLPClient` 和 store getter 返回副本，外部对象 mutation 不会污染内部 aggregate。
- 已实现 `review.approved -> accepted -> completed` 自动完成，并产生 `task.completed` audit。
- 已增强 adapter 契约：Process adapter 支持 JSON object / JSONL stdout、校验返回 `correlation_id`、未知 run 抛 `AgentAdapterError`；LangGraph config 规范化到 `configurable.thread_id`；prompt 序列化不再在有 sibling context 时丢字段。

## HLP-first API 重构

- 开始执行 HLP-first API refactor：外部产品主线收敛为 HLP 协议、SDK、adapters、host。
- `loop0` / `loop1` / `loop2` 保留为内部实现和演进分层，不再作为顶层 public API 叙事。
- 第一阶段目标是翻转顶层 `loops` 导出、移除 loop0 兼容别名、引入最小 `HLPHost`、清理 AAP public alias。
- 已实现顶层 `loops` HLP-first 导出：`HLPClient`、`HLPHost`、协议对象、store、event bus 和 adapters 可直接从 `loops` 导入；loop0 runtime 只能通过 `loops.loop0` 显式使用。
- 已新增 `HLPHost`，作为应用嵌入 HLP 的最小 host seam，统一组合 store、agent adapter、event bus 和 `HLPClient`。
- 已移除 `AAPBridge` / `InMemoryAAPBridge` public alias 与旧 `contracts.py`，下行边界统一命名为 `AgentAdapter`。
- 已同步 README、HLP spec、L1 route page、loop2 架构和 stack 架构文档，明确 L0/L1 是接入既有协议栈的介绍性路由，不是项目主线。
- 已补齐网站 HLP-first 入口：首页新增 SDK surface / `HLPHost` quickstart，Implementation Guide 增加 SDK embedding 路径，并在站点验证脚本中加入 `HLPHost` / `AgentAdapter` 断言。

## HLP 本机 CLI Adapter E2E

- 新增 `docs/plans/2026-06-23-hlp-local-cli-e2e.md`，把 Codex/Kimi/Claude Code 的真实本机 CLI 验证从原先的 fake runner adapter compatibility 中拆出来。

- 单测通过 injected runner 覆盖 Codex/Kimi/Claude 的 prompt construction、JSON extraction、correlation 保持和 structured error path；真实本机 E2E 通过 `uv run loops-hlp-local-cli-demo --adapters codex,kimi,claude` 验证。
