## HLP-first 项目定位

- 将网站和 specs 的主叙事从“三层协议栈平级定义”调整为 **HLP-first**：本项目只完整定义 Human Loop Protocol。
- 将原 L1/L0 页面改为路由说明：L1 路由到 A2A、ACP、AGNTCY 风格 mesh 或自研 runtime；L0 路由到 MCP、Agent Skills、local tools 或 function calling registry。
- 将 conformance 收敛为 HLP-compatible / HLP-integrated 两级；L1/L0 不再作为独立 Loops 协议兼容性声明，只提供 integration evidence。
- 更新站点导航、首页、overview、reading routes、integration map、contracts、HLP spec 页面、SVG 图和 OG 图，强化 HLP 是核心协议、下层是既有生态接入路线。

## HLP SDK 与端到端 Demo

- 明确 HLP SDK 是 human responsibility loop 的 control plane，不替代 OpenAI Agents SDK、Codex、Claude Code、LangGraph、CrewAI 等 agent runtime。
- 新增稳定入口 `loops.hlp.HLPClient`，让 SDK 用户不必直接调用底层 `HumanLoopOperations`。
- 将历史 `AAPBridge` 语义收敛为通用 `AgentAdapter`，并保留 `AAPBridge` / `InMemoryAAPBridge` 兼容别名。
- 新增 `FakeAgentAdapter`、`PythonCallableAgentAdapter`、`ProcessAgentAdapter` 及 OpenAI Agents SDK、OpenAI Python SDK、Codex CLI、Claude Code CLI、LangGraph、CrewAI、herms 的命名 adapter 入口。
- 新增 SDK lifecycle event、SQLite durable store、无外部依赖的 deterministic end-to-end demo。
- 以 tests-first 方式覆盖 SDK 入口、adapter correlation、事件顺序、SQLite 重启恢复和 demo 输出。
- 增强 `ProcessAgentAdapter` 为 JSON-over-stdin/stdout 执行协议，支持注入 runner、命令超时、结构化 `ProcessResult`、非零退出码/非法输出/runner exception 的 `AgentAdapterError` 包装。
- 增强 `OpenAIPythonSDKAdapter`，支持注入 OpenAI Python SDK client 并调用 `client.responses.create(...)`，同时保留无依赖 callable 形态。
- 补充 HLPClient + process adapter 的 delegate/block/resume 集成测试，确认 checkpoint 联动会穿透到 CLI/process adapter。
- 增强 `OpenAIAgentsSDKAdapter`，支持注入 `runner + agent` 并调用 OpenAI Agents SDK 的 `run/run_sync` 形态。
- 增强 `LangGraphAdapter`，支持注入 compiled graph 并调用 `ainvoke/invoke`，同时透传 HLP metadata 和可选 graph config。
- 增强 `CrewAIAdapter`，支持注入 crew 并调用 `akickoff/kickoff_async/kickoff`，同时透传 HLP metadata。
