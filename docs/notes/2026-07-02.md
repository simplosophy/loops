# 2026-07-02 变更记录

## HLP spec 连续控制扩展（0.1.0-draft → 0.2.0-draft）

### 动机

对四类 agent harness 做了系统调研：
- 编码 CLI（Codex / Claude Code / Aider / Gemini CLI）—— 30 个去重原语
- 多 agent 框架（LangGraph / AutoGen / CrewAI / OpenAI Agents SDK）—— 15 个去重原语
- 云端自主 agent（Devin / Factory / Jules / Copilot / Q / OpenHands / Replit / v0·Bolt·Lovable）—— 32 个去重原语
- 既有协议标准（MCP / A2A / OpenAI Responses / AWS Step Functions+Bedrock / AutoGen / ISO·IEEE·EU AI Act）

结论分两层：
1. **定位被验证**：没有任何现有标准把「delegation → checkpoint → review →
   ownership transfer → audit」整成一个 typed wire contract。HLP 的缝隙是真的空的。
   HLP 独有的护城河（ownership chain、protocol-level audit、artifact 版本链、
   escalation as typed act、forward-only 可重放）没人占。
2. **模型有缺口**：spec 只覆盖「agent 发起阻塞 → 人决策 → agent 恢复」，
   没覆盖真实工具里占主导的「人持续在场：随时打断、随时改方向、预授权、
   中途接管、中途改状态」。这些原语要么 spec 没有，要么被 §2.3 不可变性直接禁止。

### 修订内容（spec）

| # | 变更 | §级 | 破不破 forward-only |
|---|------|-----|---------------------|
| 1 | `task.interrupt`：人发起，`in_progress→blocked`，系统 raise `kind=interrupt` checkpoint | §3.3 / §4 / §5.1 | 不破 |
| 2 | `task.amend` + `steering_log`：append-only 方向修正，不改 spec/state | §2.3 / §3.2 / §4 / §5.1 | 不破（append-only） |
| 3 | `PermissionGrant` + `autonomy` + `Checkpoint.proposed_actions`：预授权边界 + 批量/部分审批 | §3.2 / §3.4 | 不破（新值对象） |
| 4 | `Review.kind: plan \| deliverable`；状态机加 `under_review→in_progress`（plan approved） | §3.3 / §3.6 | 不破（新枚举+新转移） |
| 5 | `CheckpointResolution.state_patch` / `edited_artifact_ref`：resume 带改过的状态 | §3.4 / §5.1 | 不破（可选字段） |
| 6 | §3.3 注：`in_progress` assignee MAY 在 take-over 期间是 human | §3.3 | 不破 |

操作数 21 → 23；状态机新增 interrupt 边与 plan-approved 回 in_progress 边；
audit action 表加 `task.interrupted` / `task.amended`；§2.3 不可变性扩充含

附录 A 时序改为含 amend/interrupt/state_patch 的扩展流；附录 B 加 0.2.0-draft 条目。

### 关键设计取舍

- **不放弃 forward-only**：转向用 append-only `steering_log`，预授权用 append-only
  `PermissionGrant`。连续控制纳入反而强化可审计性。
- **不 inflate 一等对象**：新结构都是值对象，保持「7 个一等对象」。
- **不破向后兼容**：所有新增字段可选、新枚举默认值对齐旧行为、新状态转移是新增非删除。
